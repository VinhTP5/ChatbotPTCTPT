NHÓM 1 — UNICODE NFC NORMALIZE  (đơn giản, rủi ro thấp)
══════════════════════════════════════════════════════════
Mục tiêu: chuẩn hóa tiếng Việt (NFC) trước khi split và embed
để tránh lỗi so sánh ký tự tổ hợp (ổ ≠ ổ khi dùng 2 codepoint).

Files cần sửa:
┌─────────────────────────────────────────────────────────┐
│  src/document_loader.py                                 │
├─────────────────────────────────────────────────────────┤
│  1. Thêm import unicodedata ở đầu file                  │
│  2. Thêm hàm trợ giúp:                                  │
│       def _nfc(text: str) -> str:                       │
│           return unicodedata.normalize("NFC", text or "")│
│  3. concat_pages_text(): áp dụng _nfc cho page_content  │
│     trước khi join:                                      │
│       content = _nfc((p.page_content or "").strip())    │
│  4. chunk_file(): gọi _nfc(chunk.page_content) trên     │
│     mỗi chunk SAU KHI split nhưng TRƯỚC khi gán metadata│
│  5. split_text_with_spans(): gọi _nfc(text) ở đầu hàm  │
└─────────────────────────────────────────────────────────┘
┌─────────────────────────────────────────────────────────┐
│  src/rag_engine.py                                      │
├─────────────────────────────────────────────────────────┤
│  6. Trong RAGEngine.ask(): normalize câu hỏi đầu vào   │
│       question = unicodedata.normalize("NFC", question) │
└─────────────────────────────────────────────────────────┘

Ghi chú: late_chunking.py không cần sửa vì nhận text đã normalize
từ concat_pages_text(). Query trong embed_query() được normalize
qua ask() trước khi gọi retriever.

LƯU Ý QUAN TRỌNG: Sau khi sửa xong, các collection cũ trong
chroma_db có thể bị lệch Unicode với chunk mới. Nếu muốn
nhất quán hoàn toàn, cần rebuild collection đang dùng
(bge_m3__fine__late). Tuy nhiên trong thực tế ảnh hưởng nhỏ
vì retrieval dùng cosine similarity — chỉ ảnh hưởng exact match.
→ Quyết định: KHÔNG rebuild ngay, chỉ áp dụng từ lần index mới.

══════════════════════════════════════════════════════════
NHÓM 3 — NEIGHBOR + RERANK  (phức tạp nhất, cần test kỹ)
══════════════════════════════════════════════════════════
Mục tiêu: bổ sung 2 kỹ thuật nâng cao sau retrieval:
  (a) Neighbor expansion: mở rộng context bằng cách lấy thêm
      chunk liền kề (chunk_index ± neighbor_k) trong cùng tài liệu
  (b) Reranker: dùng BAAI/bge-reranker-v2-m3 để xếp lại thứ tự
      chunks theo relevance thực sự (cross-encoder, chính xác hơn
      cosine similarity)

─────────────────────────────────────────────────────────
BƯỚC 3.1 — Tạo src/reranker.py (file mới)
─────────────────────────────────────────────────────────
  Class BGEReranker:
    __init__(model_name="BAAI/bge-reranker-v2-m3", device=None)
      · Load AutoTokenizer + AutoModelForSequenceClassification
      · Fallback gracefully nếu model chưa download

    rerank(query: str, docs: list[Document], top_n: int) -> list[Document]
      · Tạo pairs [(query, doc.page_content), ...]
      · Tokenize batch → forward pass → sigmoid score
      · Sắp xếp giảm dần theo score
      · Trả về top_n docs đã rerank

    Xử lý batch lớn: chia thành sub-batches 32 items/batch
    để tránh OOM khi docs nhiều.

  Hàm module-level:
    get_reranker(model_name=...) -> BGEReranker (singleton/cache)

─────────────────────────────────────────────────────────
BƯỚC 3.2 — Sửa src/config.py
─────────────────────────────────────────────────────────
  Thêm vào cuối phần Retrieval:
    RERANKER_MODEL = "BAAI/bge-reranker-v2-m3"
    DEFAULT_USE_RERANK = False          # opt-in, không bắt buộc
    DEFAULT_NEIGHBOR_K = 0              # 0 = tắt, 1 = lấy ±1 chunk
    DEFAULT_RERANK_TOP_N = None         # None = bằng top_k

─────────────────────────────────────────────────────────
BƯỚC 3.3 — Sửa src/rag_engine.py
─────────────────────────────────────────────────────────
  RAGEngine.__init__():
    + Thêm params: use_rerank=False, neighbor_k=0, rerank_top_n=None
    + Nếu use_rerank=True: khởi tạo BGEReranker (lazy load)
    + Lưu self._use_rerank, self._neighbor_k, self._rerank_top_n

  Thêm method _expand_with_neighbors(docs):
    · Với mỗi doc, đọc chunk_index và document_name từ metadata
    · Query Chroma với where filter = {document_name: X} để lấy
      chunk_index trong khoảng [idx-neighbor_k, idx+neighbor_k]
    · Deduplicate theo chunk_index, giữ thứ tự ban đầu
    · Trả về docs mở rộng (không thay thế, bổ sung thêm)

  Sửa RAGEngine.ask():
    1. docs = self.retriever.invoke(question)           # retrieval
    2. if self._neighbor_k > 0:
         docs = self._expand_with_neighbors(docs)       # neighbor
    3. if self._use_rerank:
         top_n = self._rerank_top_n or self._top_k
         docs = self._reranker.rerank(question, docs, top_n)  # rerank
    4. context = format_docs(docs)
    5. answer = self._answer_chain.invoke(...)

  set_retrieval_params(): thêm use_rerank, neighbor_k, rerank_top_n
